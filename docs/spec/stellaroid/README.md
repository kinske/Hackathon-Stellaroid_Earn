## How to Use This Folder
- Start with `spec-driven-development.md` for the implementation-ready plan.
- Use `product-context.md` to understand audience, positioning, constraints, and product gaps.
- Use `execution-roadmap.md` for phase order, acceptance criteria, and blockers.
- Use `seo-performance.md` when the project has SEO, performance, Core Web Vitals, canonical, metadata, or hydration work.
- Use `source-coverage.md` and `all-suggestions.md` to verify that each source file's recommendations were inspected and represented.
- Use `conflicts-and-gaps.md` before executing; it lists contradictions, missing evidence, and scope boundaries.

## North Star
Complete the credential proof loop before expanding marketplace scope: issue, verify, revoke/suspend, export, and share trusted proof pages quickly.
