# docs-new Workspace

`docs-new/` is a staging area, not the canonical product source of truth.

Use the root `README.md`, `ROADMAP.md`, `MAINTENANCE.md`, `docs/STELLAROID_TECH_CUTOVER.md`, and the current `frontend/` and `contract/` code for verified project state.

## Keep

- `bootcamp-ops/participant-checklist.md` - useful historical bootcamp operations checklist.
- `bootcamp-ops/program-flow.pdf` - keep only if it is the final participant artifact.
- `specs/stellaroid-earn-spec.md` - useful origin/spec artifact, but older than the current implementation.
- `specs/dapp-idea-prompt-template.md` - useful prompt/template artifact.

## Scratch / Research

- `research/bootcamp-mvp-strategy.md`
- `research/nft-integration-research.md`
- `specs/web3-nft-badge-spec.md`

These files include raw citation tokens, modeled planning inputs, market claims, or future-scope NFT ideas. Treat them as scratch research until citations are normalized and claims are reverified.

## Do Not Commit

- `_credentials/`
- local recovery notes
- testnet secret-key bundles
- generated screenshots or browser artifacts

The `_credentials/` folder is intentionally ignored by `.gitignore`.
